# vps-init

معالج (wizard) تفاعلي لإعداد سيرفر VPS جديد على **Ubuntu 22.04 / 24.04** بأمر واحد.

يقوم بـ:

- ✅ تفعيل الدخول عبر **SSH بكلمة المرور** — مع تجاوز إعدادات `cloud-init` التي تمنعه افتراضيًا.
- ✅ السماح بدخول **root** (اختياري).
- ✅ تعيين/تغيير **كلمة مرور** حساب.
- ✅ تغيير **منفذ SSH** — يدعم *socket activation* في أوبنتو 24 (`ssh.socket`).
- ✅ تثبيت وضبط **fail2ban** لحماية SSH من هجمات brute-force (مع حظر تصاعدي).

## التشغيل

```bash
curl -fsSL https://raw.githubusercontent.com/haydary1986/vps-init/main/setup.sh | sudo bash
```

أو (طريقة بديلة تُبقي الطرفية متصلة):

```bash
sudo bash <(curl -fsSL https://raw.githubusercontent.com/haydary1986/vps-init/main/setup.sh)
```

السكربت يقرأ مدخلاته من `/dev/tty`، لذا تعمل أسئلة الـ wizard بشكل صحيح حتى عند الاستدعاء عبر الأنبوب `curl | bash`.

## ملاحظات مهمة

- **لا تُغلق جلستك الحالية** قبل اختبار الدخول من نافذة جديدة — تحسبًا لأي خطأ.
- السكربت ينشئ **نسخة احتياطية** من `/etc/ssh/sshd_config` قبل أي تعديل، ويتحقق بـ `sshd -t` قبل إعادة التشغيل.
- إن غيّرت المنفذ والجدار الناري `ufw` مُفعّل، يُفتح المنفذ الجديد تلقائيًا.

## ⚠️ تنبيه أمني

فتح دخول root بكلمة المرور يجعل الخادم هدفًا مباشرًا لهجمات brute-force. لذلك:

- استخدم كلمة مرور قوية جدًا.
- أبقِ **fail2ban** مفعّلًا (وهو خيار افتراضي في المعالج).
- الأفضل لاحقًا: التحوّل إلى مفاتيح SSH وإعادة تعطيل الدخول بكلمة المرور.
